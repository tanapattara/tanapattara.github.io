ปรับปรุงล่าสุด: 2020-06-12

บทเรียนนี้เป็นการสร้าง รวม และส่งต่อข้อมูลของ React คอมโพเนนท์ (Component) อย่างง่าย ประกอบไปด้วยการสร้างคอมโพเนนท์ย่อจากข้อมูลวัตถุอาร์เรย์ ด้วยการทำซ้ำ ส่งข้อมูลวัตถุอาร์เรย์ไปยังคอมโพเนนท์ย่อย รวมถึงการอัพเตทข้อมูลคอมโพเนนต์ จากการเรียกใช้งานฟังชั่นข้ามคอมโพเนนท์ โดยใช้คอมโพเนนต์จากการทดลองก่อนหน้า มาประกอบเป็นคอมโพเนนท์ที่ซับซ้อน ดังรูป

  4. ติดตั้ง Visual Studio Code Extension - Simple React Snippets
  5. ติดตั้ง Visual Studio Code Extension - Prettier - Code Formatter (ตัวเลือกเพิ่มเติม)

สร้างคอมโพเนนต์

จากการทดลองครั้งก่อน React Component เราได้ทำการสร้างคอมโพเนนต์ counter และสร้างฟังชั่นสำหรับการเพิ่มตัวนับ (counter) ที่ถูกเก็บไว้ใน state ของคอมโพเนนต์ โดนเี่สร้างไฟล์ใหม่ ชื่อว่า counter.jsx ในโฟลเดอร์ src/components

counter.jsx

import React, { Component } from "react";

class Counter extends Component {
  state = {
    count: 0,
  };

  handleIncrement = () => {
    this.setState({ count: this.state.count + 1 });
  };
  render() {
    return (
      <div>
        <span className={this.getBadgeClasses()}>{this.formatCount()}</span>
        <button
          onClick={this.handleIncrement}
          className="btn btn-secondary btn-sm"
        >
          Increment
        </button>
      </div>
    );
  }

  getBadgeClasses() {
    let classes = "badge m-2 ";
    classes += this.state.count === 0 ? "badge-warning" : "badge-primary";
    return classes;
  }

  formatCount() {
    const { count } = this.state;
    return count === 0 ? "Zero" : count;
  }
}

export default Counter;

สร้างไฟล์ใหม่ เรียกว่า counterlist.jsx ไว้ในโฟลเดอร์ src/components ภายในเรียกใช้งาน คอมโพเนนต์ counter ที่เราสร้างไว้ก่อนหน้า โดยทำการ import และเรียกใช้งานเหมือนในหน้า index.js

เปลี่ยนการเรียกใช้คอมโพเนนต์

ที่ไฟล index.js เปลี่ยนคำสั่งเรียกใช้คอมโพเนนต์จากเดิม จากการเรียนกใช้ counter เป็นการเรียกใช้คอมโพเนนต์ใหม่ counterlist

index.js

//...
import Counterlist from "./components/counterlist";

ReactDOM.render(<Counterlist />, document.getElementById("root"));

ทดสอบการทำงาน

เราจะเห็นว่า counterlist ที่มีคอมโพเนนต์ counter ย่ออยู่ภายใน แสดงคอมโพเนนต์ออกมาสามคอมโพเนนต์ตามที่เราได้มีการเรียกใช้ใน counterlist คอมโพเนนต์ ทดสอบการทำงานของปุ่มกด จะเห็นว่า แต่ละคอมโพเนนต์มีการทำงานแยกกัน นั่นคือ กดปุ่มที่คอมโพเนนต์ใด ก็จะทำการเพิ่มจำนวนนับที่คอมโพเนนต์นั้น

หลังจากที่เรานำ counter ไปใส่ใน counterlist โดยเรนเดอร์คอมโพเนนต์ 3 ตัว ต่อไปลองใช้การวนซ้ำของวัตถุในการสร้าง แทนการเขียนคอโพเน้นลงไปโดยตรง (hard-code)

สร้างอาร์เรย์ใน state

  state = {
    counterlist: [
      { id: 1, value: 0 },
      { id: 2, value: 1 },
      { id: 3, value: 2 },
    ],
  };

วนซ้ำเพื่อสร้างคอมโพเนนต์

render() {
    return (
      <div>
        {this.state.counterlist.map((counter) => (
          <Counter key={counter.id} />
        ))}
      </div>
    );
  }

คำสั่ง map จะเป็นการวนซ้ำข้อมูล counterlist โดยในแต่ละคอบจะแทนข้อมูลแต่ละตัวด้วยตัวแปร counter

การวนซ้ำของ react component นั้น ทุกๆคอมโพเนนต์ที่สร้างขึ้น ต้องการ key ทำให้เราต้องใส่ค่า counter.id ลงไปเป็นค่า key

Caution: ทดสอบการสร้างคอมโพเนนต์โดยลบ key ออกจากคำสั่ง ให้เหลือเฉพาะ และลองทดสอบการทำงาน จะพบการแจ้งเดือนว่า ในลิสต์จะต้องมี key เพื่อให้สามารถอ้างถึงคอมโพเนนต์แต่ละตัวได้

แสดงผลการทำงาน

จะเห็นได้ว่า เราสามารถสร้างคอมโพเนนต์โดยการทำซ้ำจากข้อมูลของอาร์เรย์ได้ แต่หากสังเกตุจะพบว่าข้อมูล value ที่เราสร้างไว้ใน array เริ่มต้นที่ 0, 1, 2 ตามลำดับ เมื่อทดสอบการทำงาน ค่าที่พบจะเริ่มจาก 0 แต่คอมโพเนนต์ยังคงทำงานปรกติ เนื่องจากว่าขณะนี้คอมโพเนนต์ counter ยังใช้ข้อมูลจาก state ของตัวเองอยู่นั่นคือเริ่มจาก 0

หลังจากที่เราสร้างคอมโพเนนต์ย่อยโดยสร้างจากการทำซ้ำ และใส่คีย์ให้แต่ละคอมโพเนนต์เรียบร้อยแล้ว แต่เรายังไม่ได้ส่งข้อมูล value ไปยังคอมโพเนนต์ย่อย สังเกตุได้จากการทดลองครั้งที่แล้วที่ทุกครั้งค่าจะเริ่มจาก 0 ไม่ได้เริ่มจาก value ตามที่เรากำหนดไว้

เพิ่ม property ให้กับคอมโพเนนต์ Counter ที่ไฟล์ counterlist.jsx

counterlist.jsx

render() {
    return (
      <div>
        {this.state.counterlist.map((counter) => (
          <Counter key={counter.id} value={counter.value} />
        ))}
      </div>
    );
  }

ตอนนี้คอมโพเนนต์ Counter จะมี property เพิ่มขึ้นมา จากการที่เราเขียนเพิ่มลงไปที่ counterlist.jsx ทำให้คอมโพเนนต์ Counter จะสามารถเรียกใช้งานหรืออ่านข้อมูลที่ถูกใส่มาได้ โดยใช้คำสั่ง this.props

กลับไปที่ counter.jsx เพิ่มคำสั่ง this.props เพื่อเรียกใช้ property ที่ถูกส่งมาโดยทำการตั้งค่า count ที่อยู่ภายใน state ด้วยคำสั่ง

counter.jsx

  state = {
    count: this.props.value,
  };

ทดสอบการทำงาน ตอนนี้คอมโพเนนต์ counter ที่เป็นคอมโพเนนต์ย่อย ได้รับข้อมูลจาก counterlist เป็นที่เรียบร้อยแล้ว

อย่างที่เราทราบในการทดลองก่อนหน้า State จะเป็นส่วนของการเก็บข้อมูลในแต่ละคอมโพเนนท์ ซึ่งแต่ละคอมโพเนนต์จะมี state เก็บข้อมูลไม่สามารถเข้าถึงได้จากคอมโพแนนท์ภายนอก

Prop จะเป็นค่าที่เราได้รับจากคอมโพเนนท์ภายนอก เพื่อเรียกใช้ภายในคอมโพเนนท์ เราสามารถรับค่าจากภายนอกผ่าน Property ทั้งค่าเดี่ยว หรือวัตถุ รวมถึงรับฟังชั่นจากคอมโพเนนท์นอกได้อีกด้วย

ถึงตอนนี้ เราส่งค่า value จาก counterlist มายัง counter ผ่าน Prop และเก็บไว้ใน state ของคอมโพเนนท์ counter

การทำงานถึงตอนนี้ถูกต้อง เริ่มจากรับค่ามาจาก Prop แล้วแสดงผล หากมีการเปลี่ยนแปลงค่า ก็ทำการเพิ่มค่าใน State ปรกติ

แต่ค่าที่ State ที่อยู่ใน counterlist ไม่ได้มีการเปลี่ยนแปลงไปด้วย ที่เราเห็นตัวเลขเปลี่ยนแปลงนั้นเป็นตัวเลขที่มาจาก state ของ counter ต่างหาก เราจะทดลองในหัวข้อถัดไป

เราจะทำการเพิ่มปุ่มสำหรับล้างข้อมูลใน counterlist ทั้งหมด ให้กลับไปเป็น zero โดยไปเพิ่มปุ่มที่คอมโพเนนต์ counterlist

counterlist.jsx

<button className="btn-warning btn-sm m-2">Reset</button>

จะได้ส่วนติดต่อใหม่มีลักษณะดังรูป

สร้างคำสั่งสำหรับล้างข้อมูลใน state เรียกใช้ผ่าน onclick event

counterlist.jsx

<button className="btn-warning btn-sm m-2" onClick={this.handleReset}>
          Reset
        </button>

สร้างฟังชั่น handleReset สำหรับเซ็ตค่า counterlist.value ทุกตัวให้กลายเป็น 0 โดยใช้คำสั่ง map ในการเรียกข้อมูลทั้งหมดจากอาร์เรย์ออกมา และกำหนดค่าให้กลายเป็น 0

handleReset = () => {
    const _counterlist = this.state.counterlist.map((c) => {
      c.value = 0;
      return c;
    });
    this.setState({ _counterlist });
  };

บันทึกและทดสอบการทำงาน จะพบว่าเมื่อกดปุ่ม reset แล้วไม่เกิดการเปลี่ยนแปลงขึ้น ทดสอบโดยใช้ chrome extension ที่ชื่อ React Developer Tools ลงแล้ว กด F12 จะปรากฏแทบ React เราสามารถดูข้อมูลของแต่ละคอมโพเนนท์ได้

ตรวจสอบที่คอมโพเนนท์ counter ตัวแรก ได้มีการเพิ่มข้อมูล state เข้าไปเป็น 5 แล้ว หลังจากนั้นกดปุ่ม Reset แต่ค่าไม่เปลี่ยนแปลง แต่หากสังเกตุที่ข้อมูลทางด้านขวามือ จะเห็นว่า cointer:id 1, value: 0 นั่นคือมีการเปลี่ยนแปลงค่า value เป็น 0 แล้ว แต่ไม่มีการอัพเดตตัวเลขที่คอมโพเนนท์ counter

เมื่อกลับไปดูที่คอมโพเนนท์ counter เราจะพบว่า เราแสดงตัวเลขจาก state ของคอมโพเนนท์ แต่ในตอนเริ่มแรก เราได้มีการใส่ค่าจาก prop ที่รับมาลงไปใน state จากคำสั่ง

counter.jsx

  state = {
    count: this.props.value,
  };

ทั้งหมดนี้เนื่องจากเราเห็นการเปลี่ยนแปลงค่า ที่เกิดขึ้นใน state ของ counter เมื่อเราสั่งเปลี่ยนแปลงค่า state ใน counterlist เราที่ได้ส่งค่าไปที่ counter โดยใส่ข้อมูลไว้ที่ state คำสั่ง state นี้จะถูกเรียกเพียงครั้งแรกครั้งเดียว ในตอนสร้างคอมโพเนนท์ ทำให้เราไม่สามารใส่ค่าใหม่จากการ reset ค่าของเราใหม่ได้

จากการที่เราส่งข้อมูล id และ value ไปยังคอมโพเนนท์ย่อยและเก็บไว้ที่ state ของแต่ละคอมโพเนนต์ เราจะส่งข้อมูลไปทั้งหมดเป็นวัตถุ counter ทั้งวัตถุแทน และแสดงผลผ่านการเรียกใช้ property ทีเดียวไม่ต้องเก็บลง state

การส่งข้อมูล

ที่ counterlist เปลี่ยน property ใหม่จากการส่งข้อมูลแบบเดิมทีละตัว value={counter.value} ให้ส่งข้อมูลวัตถุ counter ไปทั้งก้อน counter={counter}

counterlist.jsx

<div>
   <button className="btn-warning btn-sm m-2" onClick={this.handleReset}>
          Reset
   </button>
   {this.state.counterlist.map((counter) => (
     <Counter
       key={counter.id}
       //sent a single value
       //value={counter.value}
       //sent all obj
       counter={counter}
     />
   ))}
</div>

การรับข้อมูล

หลังจากเราเปลี่ยน property ของ counter แล้ว การรับข้อมูลจะเปลี่ยนไป จากการรับแยกตาม property แต่เป็นการเรียกใช้ property ของก้อนข้อมูลวัตถุ ใหม่

ลบ state ของคอมโพเนนท์ counter ออก เพราะเราจะใช้ข้อมูลที่เราได้รับจาก counterlist ผ่าน property อัพเดตส่วนการแสดงผลของฟังชั่น ให้เรียกผ่าน this.props แทน

counter.jsx

  getBadgeClasses() {
    let classes = "badge m-2 ";
    classes +=
      this.props.counter.value === 0 ? "badge-warning" : "badge-primary";
    return classes;
  }

  formatCount() {
    const { value } = this.props.counter;
    return value === 0 ? "Zero" : value;
  }

เนื่องจากเราใช้ข้อมูล counterlist ที่เก็บไว้ใน state ของคอมโพเนนท์ counterlist ดังนั้น การที่ปุ่มเพิ่มค่า ที่อยู่ในคอมโพเนนท์อื่น จะมาทำการเปลี่ยนแปลงค่านอกคอมโพเนนท์ counterlist ย่อมไม่สามารถทำได้ ข้อมูลของคอมโพเนนท์ใด ย่อควรให้คอมโพเนนท์นั้นเป็นคนจัดการ ดังนั้น เราจะทำการเพิ่มค่าให้กับตัวแปรที่คอมโพเนนท์ counterlist

    console.log(counter);
};

หลังจากนั้นทำการส่งฟังชั่น handleIncrement ไปที่คอมโพเนนท์ counter ผ่าน property

counterlist.jsx

{this.state.counterlist.map((counter) => (
          <Counter
            key={counter.id}
            counter={counter}
            onIncrement={this.handleIncrement}
          />
        ))}

เปลี่ยนการทำงานของปุ่ม Increment ที่คอมโพเนนท์ counter ให้เรียกใช้ฟังชั่นจาก property แทน

counter.jsx

<button className="btn btn-secondary btn-sm"
        onClick={() => this.props.onIncrement(this.props.counter)}>
   Increment
</button>

ทดสอบการทำงาน

จากการทดสอบผ่าน console.log เมื่อกดปุ่ม increment ที่คอมโพเนนท์แรก ที่ console จะแสดงข้อมูลคอมโพเนนท์ได้ถูกต้องตามที่เรากด นั่นก็คือ id: 1 และ value:0 เนื่องจากว่า ที่ฟังชั่น onClick ที่คอมโพเนนท์ counter เราเรียกใช้ onIncrement จาก property และส่งวัตถุ counter กลับมาทั้งก้อน จากคำสั่ง this.props.onIncrement(this.props.counter)

ปรับปรุง onIncrement()

ที่ฟังชั่น onIncrement เรารับค่า counter มาโดยเราทดสอบแสดง console.log(counter); แล้วให้ข้อมูลที่ถูกต้อง ต่อไปเราจะทำการอัพเดตค่าให้กับ counterlist ของเราโดยใช้คำสั่ง

หลังจากนั้นทำการส่งฟังชั่น handleIncrement ไปที่คอมโพเนนท์ counter ผ่าน property

counterlist.jsx

  handleIncrement = (counter) => {
    // clone state.counter
    const counterlist = [...this.state.counterlist];
    // get array index by compair between counterlist and counter
    const index = counterlist.indexOf(counter);
    // clone data from counter to selected index
    counterlist[index] = { ...counter };
    // update value by 1
    counterlist[index].value++;
    // setState
    this.setState({ counterlist });
  };

ทดสอบการทำงาน

ขอแสดงความยินดีด้วย คุณได้สร้างเว็บแอพพลิเคชั่นโดยใช้ React JS สำเร็จแล้ว

การทดลองเราได้รวมคอมโพเนนท์ย่อย และเรียกใช้เป็นคอมโพเนนท์เดียวกัน ผ่านคอมโพเนนท์ counterlist เราได้สร้างคอมโพเนนท์จากข้อมูลอาร์เรย์ ทำซ้ำตามจำนวนข้อมูล และส่งค่าไปยังคอมโพเนนท์ย่อย รวมถึงฟังชั่นเพื่อให้มีการอัพเดตข้อมูลระหว่างคอมโพเนนท์ด้วยกัน อีกทั้งยังใช้เครื่องมือในการตรวจสอบค่าภายในแต่ละคอมโพเนนท์ โดยใช้เครื่องมือ React Development Tools
