ปรับปรุงล่าสุด: 2020-06-12

React ถูกพัฒนาขึ้นโดย Facebook ในปี 2011 และเป็น JS Framework ที่นิยมอย่างรวดเร็ว ด้วยความสามารถในการแยกส่วนประกอบของเว็บออกเป็นคอมโพเนนต์ (Component) หรือแบ่งส่วนต่างๆของส่วนติดต่อผู้ใช้ออกเป็นส่วนย่อย และแต่ละส่วนของคอมโพเนนต์ยังสามารถแยกย่อยลงไปได้อีก การเปลี่ยนแปลงข้อมูลและอินเตอร์เฟสผ่าน Visual DOM

เราสามารแยกพัฒนาแต่ละคอมโพเนนต์ แล้วนำมาประกอบกันเป็นส่วนติดต่อผู้ใช้ที่ซับซ้อนได้

แต่ละคอมโพเนนต์ เป็น javascript class ที่ประกอบไปด้วย state และ render

  • State คือ ข้อมูลที่คอมโพเนนต์ใช้ในการทำงาน
  • Render คือส่วนที่บอกว่าหน้าตาของคอมโพเนนต์หน้าตาเป็นอย่างไร ส่วนส่งออกคือ React Element

React Element หรือ Virtual DOM เป็น JS Element ที่ทำหน้าที่แสดงผลการทำงานในรูปแบบของ DOM เมื่อมีการเปลี่ยนแปลง state ของคอมโพเนนต์ เราจะได้ React Element ตัวใหม่ React จะทำการเปรียบเทียบกับคอมโพเนนต์เดิม ทำให้ทราบว่ามีการเปลี่ยนแปลงและทำการเปลี่ยนแปลง Real DOM เพื่อทำการปรับปรุงส่วนติดต่อผู้ใช้ในบราวเซอร์

บทเรียนนี้เป็นการสร้าง React คอมโพเนนท์ (Component) อย่างง่าย ประกอบไปด้วยรูปภาพ ข้อความ และปุ่มกด รวมกับการใช้งาน CSS Frameworks ตกแต่งคอมโพเนนท์ให้ดูสวยงาม

React เป็นการรวมกันของคอมโพเนนต์ย่อยๆ รวมกันเป็นหน้าเว็บ โดยแต่ละคอมโพเนนท์จะมีหน่วยความจำและฟังชั่นของตัวเอง การทดลองนี้ เราจะทดลองสร้างคอมโพเนนต์อย่างง่าย

สิ่งที่จะได้จากบทเรียน

  • การสร้าง React Component
  • การเรียกใช้และปรับปรุง Attribute ของแต่ Element
  • การเข้าถึง State ของ Component

สร้างโปรเจคใหม่

การสร้างโปรเจค react จะใช้คำสั่งจาก command line ประกอยด้วบสองส่วน ส่วนแรกคำสั่งสร้าง และตามด้วยชื่อโปรเจค ในตัวอย่าง เป็นการสร้างโปรเจคที่ชื่อว่า react-app

create-react-app counter-app

React จะสร้างโฟลเดอร์ โปรเจค ณ ตำแหน่ง ปัจจุบันที่เราอยู่ในคำสั่ง command line

Note: เลือกตำแหน่งที่ต้องการสร้างโปรเจค เช่น C:\Sources แล้วทำการเปิด PowerShell โดยการกด Shift + Right Click และเลือก Open PowerShell window here

Error: หากไม่สามารถสร้างโปรเจคด้วยคำสั่ง create-react-app ได้ และเกิดข้อผิดพลาด ..\create-react-app.ps1 cannot be loaded because running scripts is disabled on this system.

ให้ทำการเปิด PowerShell แบบ Administrator (Run as Administrator) และใส่คำสั่ง set-executionpolicy remotesigned ตามด้วย Y เพื่อเป็นการแก้ปัญหาการสร้างโปรเจคไม่ได้ด้วยคำสั่ง create-react-app

ในหน้า command line จะมีคำสั่งที่ให้มาในการจัดการโปรเจคของเรา

  • npm start สำหรับเปิดการทำงานของเซิรฟเวอร์จำลอง
  • npm run build การทำให้แอพของเป็นไฟล์สำหรับนำไปใช้งานบนโปรดักส์ชั่นเซิรฟเวอร์
  • npm test การเปิดการทดสอบการเทส
  • npm run eject การติดตังไปยังเซิรฟ์เวอร์

หลังจากสร้างโปรเจคแล้ว เราต้องเข้าไปยังโปรเจคของเราก่อนโดยใช้คำสั่ง cd react-app จะเป็นการเข้ามายังโปรเจคที่ได้สร้างขึ้น เราสามารถเปิดโฟลเดอร์ปัจจุบันโดย Visual studio Code จาก command line โดยใช้คำสั่ง

Note: เราสามารถหยุดการทำงานของเซิรฟ์เวอร์โดยการกด crtl + c ที่ command line

ที่ Visual Studio Code เปิด Terminal และลง bootstrap โดยใช้คำสั่ง

npm i bootstrap

ทำการเพิ่ม bootstrap เข้าไปยังแแอพของเรา โดยเปิด index.js และเพิ่มคำสั่ง

index.js

...
import "bootstrap/dist/css/bootstrap.css";
...

Note: เราสามารถเปิดไฟล์ใน Visual Studio Code อย่างรวดเร็วโดยการกดปุ่มลัด ctrl + p จะปรากฎช่องขึ้นมา ให้พิมพ์ชื่อไฟล์ที่เราต้องการ เช่น index.js แล้วกด enter จะทำให้เราเปิดไฟล์ได้โดยที่ไม่ต้องไปเปิดหาที่แทบด้านซ้ายมือ

ทดสอบการทำงานด้วยคำสั่ง npm start จะพบว่าตัวหนังสือของเว็บได้เปลี่ยนไปเนื่องจากเราได้นำ boostrap มาใส่ลงไปในแอพพลิเคชั่นของเราแล้ว

เข้ามาดูภายในโปรเจค จะประกอบไปด้วยโฟลเดอร์หลัก อยู่ 3 โฟลเดอร์

  • node_modules สำหรับเก็บ third-party library รวมถึง react ด้วย
  • public สำหรับเก็บ public asset ไฟล์ต่างๆ เช่น ไอคอน รวมถึง index.html
  • src สำหรับเก็บ react component เริ่มแรก จะมีแค่ app.js component

ที่โฟลเดอร์ public จะมีไฟล์ index.html อยู่ ประกอบไปด้วย HTML และ metadata ของเว็บไซด์ ภายในแท็ก body จะมี

อยู่ตัวหนึ่งมีไอดีคือ root

โฟลเดอร์ src ไฟล์ index.js ประกอบไปด้วยคำสั่ง import เพื่อนำเข้าคลาส และคอมโพแนนต์จะใช้งานของ react

คำสั่ง ReactDOM.render() เป็นทำสั่งสร้างคอมโพแนนต์ โดยในคำสั่งเป็นการสร้างคอมโพแนนต์ที่ชื่อว่า โดยนำคอมโพแนนต์ที่สร้างไปใส่ที่อิลิเม้นท์ไอดีว่า root

index.js

import React from 'react';
import ReactDOM from 'react-dom';
import './index.css';
import App from './App';
import * as serviceWorker from './serviceWorker';

ReactDOM.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
  document.getElementById('root')
);

serviceWorker.unregister();

มาดูที่คอมโพแนนต์ รายละเอียดภายในคอมโพแนนต์ที่ไฟล์ App.js

App.js

import React from 'react';
import logo from './logo.svg';
import './App.css';

function App() {
  return (
    <div className="App">
      <header className="App-header">
        <img src={logo} className="App-logo" alt="logo" />
        <p>
          Edit <code>src/App.js</code> and save to reload.
        </p>
        <a
          className="App-link"
          href="https://reactjs.org"
          target="_blank"
          rel="noopener noreferrer"
        >
          Learn React
        </a>
      </header>
    </div>
  );
}

export default App;

เริ่มจาก import เป็นการนำเข้า component อื่นเข้ามาใช้ในคอมโพเนนต์

ที่คอมโพแนนต์นี้ได้นำเข้าไฟล์ app.css เข้ามาใช้จากคำสั่ง import './App.css';

ตามด้วย สร้างฟังชั่น App() ภายใน ได้สร่งข้อมูลคล้ายๆ html กลับไป รูปแบบภาษานี้เราเรียกว่า JSX หรือ JavaScript Xml รูปแบบการเขียนคล้าย html แต่มีบางส่วนที่ไม่เหมือนกัน เช่น JSX ใช้ className แทน class attribute ใน html เป็นตัน

JSX ที่เห็นนี้ จะรวมเป็น App คอมโพเนนต์ และจะถูก Babel นำไปแปลงเป็น HTML ที่ browser เข้าใจได้ และแสดงผลที่หน้าแรกที่เราเห็นจากการทดสอบการทำงาน

ทดลองเปลี่ยนข้อความภายใน JSX ให้เป็น

    <div className="App">
      <header className="App-header">
        <img src={logo} className="App-logo" alt="logo" />
        <p>
          Hello React JS
        </p>
      </header>
    </div>

Note: หากเราทำการแก้ไข code ระหว่างที่มีการทำงานของเซิรฟเวอร์ ด้วยคำสั่ง npm start เมื่อเราแแก้ไขและมีการบันทึก (Save) หน้าเว็บจะทำการเปลี่ยนแปลงให้อัตโนมัติโดยไม่ต้องสั่ง F5 หรือปิดเปิดเซิรฟ์เวอร์ใหม่ เราเรียกการทำงานนี้ว่า Hot Reload

สร้างคอมโพเนนต์ใหม่

ไปที่โโฟลเดอร์ src ทำการสร้างโฟลเดอร์ใหม่ให้อยู่ภายใต้โฟลเดอร์ src ชื่อว่า components เราจะใช้โฟลเดอร์นี้ทำการเก็บคอมโพเนนต์ที่เราสร้างขึ้นไว้ภายใน

สร้างไฟล์ใหม่ ชื่อว่า counter.jsx

counter.jsx

import React, { Component } from 'react';

class Counter extends Component {

    render() { 
        return <h1>Hello World</h1>;
    }
}
 
export default Counter;

เราได้ทำการสร้างคอมโพเนนต์คลาสที่ชื่อว่า Counter โดยคอมโพเนนต์นี้ทำหน้าที่แสดงผล Hello world โดยใช้แท็ก H1 ต่อไปเราจะนำคอมโพเนนต์นี้ไปใส่ลงใสแอพของเรา

นำเข้าและแสดงคอมโพเนนต์

ที่ไฟล index.js ทำการเพิ่ม คำสั่ง import และ เปลี่ยนอากิวเมนท์ตามตัวอย่าง

index.js

//...
import Counter from "./components/counter";

ReactDOM.render(<Counter />, document.getElementById("root"));

เพิ่มคำสั่ง import ต่อท้ายคำสั่ง import อื่นๆ กำหนดชื่อคอมโพเนนต์ และที่อยู่ของไฟล์ ตามที่เราได้สร้างไว้ในโฟลเดอร์ components และไฟล์ counter ตามลำดับ

แล้วทำการเปลี่ยนอากิวเมนท์ render ให้เป็นแท็ก

React จะทำการเปลี่ยนแท็ก ให้เป็นคอมโพเนนท์ที่เราอ้างอิงถึงในตอน import นั่นคือไฟล์ counter.jsx ที่อยู่ในโฟลเดอร์ components

ทดสอบการทำงาน ตอนนี้ หน้าหลักได้เปลี่ยนเป็น Hello world คอมโพเนนต์เค้าเตอร์ที่เราได้สร้างขึ้นแล้ว

ที่ไฟล์ counter.jsx ทดลองเพิ่มแท็กอื่นๆลงไป ดังตัวอย่าง

counter.jsx

import React, { Component } from 'react';

class Counter extends Component {

    render() { 
       return <h1>hello world</h1><button>Add</button>;
    }
}
 
export default Counter;

สังเกตุว่าตอนนี้เรามี element อยู่สองอันนั่นคือ h1 กับ button

React จะแจ้งข้อผิดพลาดให้เราทราบด้านหลังสุด เนื่องจากว่า jsx ต้องมี parent element หรืออิลิเมนท์ตัวบนสุดตัวเดียว ทางแก้คือ เราจะทำให้อิลิเมนท์ทั้งสองอยู่ภายใต้อิลิเมนท์เดียวโดยการใส่

ครอบทับดังตัวอย่าง

counter.jsx

import React, { Component } from 'react';

class Counter extends Component {
  render() {
    return (
      <div>
        <h1>hello world</h1>
        <button>Increment</button>
      </div>
    );
  }
}

export default Counter;

ทดสอบการทำงาน จะเห็นว่า มีปุ่ม Incrementเพิ่มมาแล้ว ลองดูที่ dev tools จะเห็นว่า ทั้งสองอยู่ภายใต้

หากเราไม่ต้องการให้คอมโพเนนต์ของเราอยู่ภายใต้ div ที่ไม่ได้ทำอะไรเลย (เช่น ไม่มีคลาส หรือไอดีใดๆ) เราสามารถใช้ แทนทีการใช้

เปล่าๆได้ ตัวอย่างเช่น

counter.jsx

import React, { Component } from 'react';

class Counter extends Component {
  render() {
    return (
      <React.Fragment>
        <h1>hello world</h1>
        <button>Increment</button>
      </React.Fragment>
    );
  }
}

export default Counter;

ทดสอบการทำงาน ตรวจสอบที่ dev tool จะเห็นว่าไม่มี <div> ครอบทับแท็กทั้งสองแล้ว เราสามารถเลือกใช้ได้ตามสถานะการณ์

ที่ counter.jsx ทำการเพิ่ม state ซึ่งมีหน้าที่เก็บข้อมูลของแต่ละคอมโพเนนต์เอาไว้ โดยสร้างตัวแปรตัวแรกให้ชื่อว่า count และกำหนดค่าเท่ากับ 0

ที่อิลิเมนท์ h1 เปลี่ยนเป็น span พร้อมกับเรียกใช้ตัวแปร count ที่สร้างขึ้นภายใน { } (เครื่องหมาย Braces หรือ curly bracket) โดยใช้คำสั่ง this.state.count ทำให้เราสามารถแสดงผลอิลิเมนท์แบบไดนามิกส์ได้

counter.jsx

...
class Counter extends Component {
  state = { count: 0 };

  render() {
    return (
      <div>
        <span>{this.state.count}</span>
        <button>Increment</button>
      </div>
    );
  }
}
...

จะเห็นว่า ข้อความ hello world ได้ถูกเปลี่ยนเป็น 0 เนื่องจากค่าตัวแปร count มีค่าเท่ากับ 0 ตามที่เราได้ประกาศไว้ใน state ทดลองเปลี่ยนค่า count เพื่อดูการเปลี่ยนแปลงของหน้าเว็บ

ภายใน { } เราสามารถคำนวนตัวเลข หรือตัวแปรได้ เช่น

{2 + 2}

สร้างเมธอดย่อย

ต่อไปเราจะเพิ่มเงื่อนไขการแสดงผลให้กับคอมโพเนนท์ หากว่าตัวแปรเท่ากับ 0 ให้แสดงข้อความว่า Zero หากไม่ใช่ให้แสดงตัวเลขปรกติ ให้เราเพิ่มเมธอดให้กับคอมโพเนนต์ ตั้งชื่อเมธอดว่า formatCount() และเรียกใช้เมธอดเพื่อแสดงผลผ่านเครื่องหมาย { } ด้วยคำสั่ง {this.formatCount()}

counter.jsx

class Counter extends Component {
  state = { count: 0 };

  render() {
    return (
      <div>
        <span>{this.formatCount()}</span>
        <button>Increment</button>
      </div>
    );
  }

  formatCount() {
    return this.state.count === 0 ? "Zero" : this.state.count;
  }
}

จะเห็นได้ว่าเราเรียกใช้ state เป็นวัตถุ เราสามารถสร้างวัตถุให้กับ state.count ได้ โดยใช้คำสั่งสร้างวัตถุ แทน state.count โดยทำการปรับปรุงเมธอด formatCount() ดังนี้

counter.jsx

  formatCount() {
    const { count } = this.state;
    return count === 0 ? "Zero" : count;
  }

JSX ทำให้เราสามารถส่งออกอิลิเมนท์ไปได้เหมือนกับตัวแปรตัวแปรหนึง ทดลองเปลี่ยนผลลัพท์ Zero เป็นคำสั่ง

counter.jsx

  formatCount() {

  }

เราจะได้อิลิเมนท์ h4 มาแสดงผลแทนข้อความธรรมดา

เพิ่มตัวแปร imageUrl ใน state และเพิ่มอิลิเมนท์สำหรับแสดงรูปภาพด้วยแท็ก

เพิ่มคลาสทริบิวท์ให้กับอิลิเมนท์

เพิ่มคลาสให้กับอิลิเมนท์ของเรา โดยแทรก className ลงไปในแอททริบิวท์ในแต่ละอิลิเมนท์ โดยนำเอาคลาสที่มีใน bootstrap มาใช้ในการแสดงผล หรือเราจะใช้คลาสที่เราสร้างขึ้นเองใน css ในการทำเว็บจริงก็ได้ ตัวอย่างใช้คลาสจาก bootstrap ที่ได้ติดตั้งและนำเข้าไว้ตั้งแต่ตอนแรกของบทเรียน

counter.jsx

Note: เราไม่สามารถใช่คำสั่ง class เฉยๆได้เนื่องจากเป็นคำสงวนใน JS

สร้างตัวแปรขึ้นมาเพื่อเก็บค่า css class ตรวจสอบเงื่อนไข และนำไปแสดงผล

Update State

เมื่อเข้าถึง this.state ได้แล้ว เรามาเปลี่ยนแปลงค่าโดยใช้คำสั่ง

counter.jsx

...
 handleIncrement = () => {
    this.state.count++;
    console.log("Increment button clicked", this.state.count);
  };
...

ทดสอบการแสดงผล

จะเห็นว่าเมื่อกดแล้วไม่มีการเปลี่ยนแปลงใดๆ แต่หากดูใน console แล้ว count จะมีการเปลี่ยนแปลงค่าทุกครั้งที่มีการกด ในทางเทคนิคแล้ว ตัวแปรมีการเปลี่ยนแปลงค่า แต่การเปลี่ยนแปลงแบบนี้ React ไม่ทราบถึงการเปลี่ยนแปลงของคอมโพเนนต์ ทำให้ react ไม่ได้เปลี่ยนแปลง view ตาม เราแก้ไขได้จากฟังชั่นที่สืบทอด (Inherit) มาจาก Component แม่ เรียกใช้เมธอด this.setState() โดยเป็นการบอก React ว่า state ได้มีการเปลี่ยนแปลงไปแล้วให้ทำการอัพเดต view ด้วย

counter.jsx

...
  handleIncrement = () => {
    this.setState({ count: this.state.count + 1 });
  };
...

ทดสอบการทำงาน จะพบว่าเมื่อกดปุ่ม Increment จะมีการเพิ่มจำนวนขึ้นไปทีละ 1

Note: หากเปิด dev tool ใน Elements tab ที่คอมโพเนนต์ที่เราสร้างขึ้น เมื่อมีการกดปุ่มเพื่อเพิ่มค่า และมีการเปลียนแปลง state ตามคำสั่ง this.setState() React จะทราบว่ามีการเปลี่ยนแปลงเกิดขึ้นที่คอมโพเนนต์ใดคอมโพเนนต์หนึ่ง และตรวจสอบว่ามีอะไรเปลี่ยนแปลงค่าไปบ้าง ในที่นี้ มีการเปลี่ยนแปลงค่า count และนำมาแสดงผลที่ ทำให้ React เปลี่ยนแปลงเฉพาะส่วนของ ไม่ได้เปลี่ยนแปลงส่วนอื่นๆ
